Description

You still have a lot to say to your clients. Every educational product should be fully represented. Let's add two more pages. One will contain information about the course content, teaching methods, duration, and staff. The other page will contain something like CVs of your best mentors, administrators, moderators, etc. to guide your student through the program to the mountains of knowledge.

You already know how to create templates, so add two more to the templates directory. Think of interesting headings and add any information that would be useful for your clients. Also, we are going to display information from our models. To access the fields of the necessary model you can use the so-called dot notation â€” {{ model name.field name }}. This page will have course_details as the title. Do the same thing for the page on teachers (teacher_details) but with the Teacher model.

To navigate to these brand-new pages, users need links to them on the previous pages. They are main.html for course_details and course_details.html for teacher_details.

The next step is to add two view classes for each page in views.py. Create the template_name and model fields in them.

Finally, pay attention to the paths' endings in urls.py. Use regular expressions to address the <pk> of each item in the database.

If you can easily switch from one page to another, let's continue with the next stage.

Objectives
  • Create new HTML templates for the course and teacher details pages;
  • Add links for new pages;
  • Add views for new pages
  • Use regexes in the URL mapper to adjust it.
