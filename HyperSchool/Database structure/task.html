Description

You have a good selection of educational materials that you want to share. So you decided to run an online school. Of course, you need a site where students could find a course and get information about the curriculum, price, duration, goals, and professors. It's also a good idea to monetize it in the future.

Let's start with building a database structure for our project. First of all, we need to start the basic Django project structure HyperSchool and create the app schedule inside:

python manage.py startapp schedule

Don't forget to add it to settings.py.

After this, create the database models. Throughout our project, we will require three models: Course, a model that describes our educational program, Student stores the information about our clients, and Teacher — it will give us information about the staff of the online school.

Then, add these models to models.py of our application schedule. The Teacher model should have the following fields:

  • name — a string with a max length of not less than 50;
  • surname — also, a string;
  • age — an integer value;
  • about — text to represent experience and achievements.

The Course model should have the following fields:

  • title — a string with a max length of more than 100 (let's say 255 — this length is often used);
  • info — also a string, should be longer, as a description, of course, may differ;
  • duration_months — an integer value. You may use other kinds of fields with methods to convert it or simplify input;
  • price
  • teacher — ManyToMany connection to our Teacher model.

And the model Student with fields:

  • name — a string with a max length of more than 50;
  • surname — also a string;
  • age — an integer value;
  • course — ManyToMany connection to the Course model.

The next step is to populate the new site with data using the Django admin panel. Use the default settings of the project with the predefined SQLite database. Write the Course, Student, and Teacher models into models.py, register them in admin.py, and migrate them to the database.

To see what we get on the page, let's add some courses to the database through the admin panel. You can reach it by the following URL when your server is started with the python3 manage.py runserver: 

https://127.0.0.1:8000/admin/

Create a superuser in the terminal and login properly on the admin page. If your solution is correct, the required models will run in our project. Well done! Let's add some courses, teachers, and students in the next stage!

Objectives

To pass this stage, you need to do the following:

  • Create a skeleton of the new app with the basic Django project;
  • Create the database models on par with the description above;
  • Use the Django admin panel and populate the new site with data.
